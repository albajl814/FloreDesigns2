Demostración de API de Búsqueda

Actividad 2 — Herramientas de Búsqueda

Comparativa de tres herramientas de búsqueda ampliamente usadas en desarrollo web.

1. Algolia

Características

  • Motor de búsqueda como servicio (SaaS)
  • Búsqueda en tiempo real con latencia menor a 100ms
  • Tolerancia a errores tipográficos (typo-tolerance)
  • API REST con SDKs para múltiples lenguajes
  • Panel de control visual para configurar índices

Ventajas

  • Implementación rápida sin infraestructura propia
  • Resultados relevantes desde el primer carácter
  • Escalable automáticamente según tráfico
  • Plan gratuito disponible para proyectos pequeños

Casos de uso

  • Buscadores de e-commerce (productos, filtros)
  • Búsqueda de documentación técnica
  • Autocompletado en formularios de registro
  • Catálogos de moda como el de Floré Designs
Ejemplo de petición:
GET https://YourAppID.algolia.net/1/indexes/vestidos/query
Body: { "query": "novia encaje", "filters": "categoria:novia" }

2. Elasticsearch

Características

  • Motor open-source basado en Apache Lucene
  • Búsqueda de texto completo (full-text search)
  • Soporte para búsquedas distribuidas en clústeres
  • API REST con respuestas en JSON
  • Análisis y agregaciones de datos en tiempo real

Ventajas

  • Completamente gratuito y de código abierto
  • Altamente escalable para grandes volúmenes de datos
  • Control total sobre la infraestructura y los datos
  • Soporte nativo para múltiples idiomas

Casos de uso

  • Análisis de logs de servidor (stack ELK)
  • Buscadores corporativos internos
  • Plataformas con millones de registros
  • Motores de recomendación de productos
Ejemplo de petición:
GET http://localhost:9200/vestidos/_search
Body: { "query": { "match": { "descripcion": "encaje novia" } } }

3. Google Custom Search API

Características

  • API de Google para búsquedas en sitios específicos
  • Indexación automática por el motor de Google
  • Resultados con título, snippet y URL
  • Filtros por tipo de archivo, idioma y fecha
  • Integración con Google Cloud Console

Ventajas

  • Respaldada por la infraestructura de Google
  • Sin necesidad de mantener un índice propio
  • 100 consultas gratuitas por día
  • Resultados de alta calidad y relevancia

Casos de uso

  • Buscador interno de sitios web o blogs
  • Portales de noticias con búsqueda de artículos
  • Sitios educativos con catálogos de contenido
  • Búsqueda en múltiples dominios relacionados
Ejemplo de petición:
GET https://www.googleapis.com/customsearch/v1
?key=API_KEY&cx=SEARCH_ENGINE_ID&q=vestido+novia

Por qué se implementó una API REST propia simulada en este proyecto

Las tres herramientas anteriores requieren cuentas externas, claves de API o servidores dedicados. Para este proyecto académico se optó por una API REST propia simulada en PHP que replica el mismo contrato de interfaz (endpoint, método GET, respuesta JSON) sin depender de servicios de terceros, lo que permite demostrar el concepto de integración de APIs de forma completamente local y reproducible.

Informacion de la API

Tipo de API: REST API (Simulada)
Endpoint: /api/search-api.php
Método HTTP: GET
Content-Type: application/json
Parámetros aceptados:
  • q - Término de búsqueda (string)
  • categoria - Categoría del vestido (string)
  • precio_min - Precio mínimo (number)
  • precio_max - Precio máximo (number)
  • evento - Tipo de evento (string)
